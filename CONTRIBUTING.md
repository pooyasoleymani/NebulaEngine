## How to Contribute
- Fork the repository and create your branch from `main`.
- Ensure code builds and tests pass (`cmake --build build --target test`).
- Add or update documentation when changing APIs.
- Submit a pull request with a clear description of your changes.

## Coding Standards
- Use C++17 and follow modern best practices.
- Write unit tests with Catch2 for new features.
- Run benchmarks when optimizing performance.

## Reporting Issues
- Use GitHub Issues for bug reports and feature requests.
- Include steps to reproduce and expected behavior.
